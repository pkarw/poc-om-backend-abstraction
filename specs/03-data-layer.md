# Data Layer Requirements — Entities, Migrations, Multi-Tenancy, Custom Fields, Query Engine, Commands

> Derived from upstream commit adc9da27759e357febe9ed8d4b7182040d127349. Source analysis: ../upstream/analysis/03-data-layer.md

## Scope

This spec covers everything a ported module observes through PostgreSQL and the read/write abstractions:

- Entity/table conventions (naming, PKs, timestamps, soft delete, tenancy columns).
- Per-module migration workflow and migration-tracking tables.
- Multi-tenancy model (directory module schema, `OrganizationScope` resolution, three enforcement layers).
- Custom entities & custom fields (`entities` module tables, EAV writes, JSONB doc storage, `ce.ts` installation).
- Query engine contract (`QueryEngine.query`), the `query_index` projection tables, indexed/fallback read paths, and write-side index maintenance.
- Command pattern for writes (pipeline, scope guards, atomic flush, action logs, undo/redo).

Out of scope: HTTP routing/auth token mechanics (02, 05), event/queue transport (04), module discovery (01). Event *names and payloads* emitted by the data layer are contracts here; their delivery is spec 04.

Because the database is shared infrastructure (a Node deployment and a ported deployment may point at the same Postgres), **all DDL names, column types, and write-side conventions below are wire contracts**, not internal details.

## Requirements

### Schema conventions

- **DATALAYER-R1** (MUST) Every persisted table uses an explicit snake_case, plural table name (e.g. `customer_entities`, `users`, `tenants`). Ports of existing modules MUST reproduce the upstream table name exactly.
- **DATALAYER-R2** (MUST) Primary keys are `id uuid DEFAULT gen_random_uuid()`. The uuid is generated by Postgres on insert; application code MUST NOT pre-assign ids except when restoring a record with its original id (undo/redo).
- **DATALAYER-R3** (MUST) Column names are snake_case. String columns are `text` (never varchar); other types map to Postgres-native `boolean`, `int`, `float`/`double precision`, `uuid`, `jsonb`/`json`, `timestamptz`.
- **DATALAYER-R4** (MUST) `created_at` and `updated_at` are set application-side (current timestamp at create/update); raw SQL paths use `now()`. There are no DB triggers for timestamps, and a port MUST NOT add them (they would fire on rows written by other runtimes and change observable values).
- **DATALAYER-R5** (MUST) Soft delete is a nullable `deleted_at` timestamptz column. Deleting sets `deleted_at = now()`; every read path excludes rows with non-null `deleted_at` unless the caller passes `withDeleted: true`. There is no global ORM filter to rely on — the exclusion is applied explicitly wherever the column exists.
- **DATALAYER-R6** (MUST) Tenancy columns are named exactly `tenant_id` and `organization_id`, type `uuid`, as plain scalar columns (no FK constraints), except `organizations.tenant_id` which is a real FK to `tenants`. On cross-cutting tables (`custom_field_defs`, `entity_indexes`, etc.) they are nullable and NULL means "global scope"; on business tables they are generally NOT NULL.
- **DATALAYER-R7** (MUST) Entity identifiers are strings `'<module>:<entity>'` matching `/^[a-z][a-z0-9_]*:[a-z][a-z0-9_]*$/`. A port MUST validate this shape at API boundaries and MUST use these ids in `custom_field_defs.entity_id`, `custom_field_values.entity_id`, `entity_indexes.entity_type`, `custom_entities_storage.entity_type`, etc.
- **DATALAYER-R8** (SHOULD) Indexes carry explicit snake_case names matching upstream (e.g. `cf_defs_entity_key_scope_idx`, `organizations_tenant_slug_uniq`). Names of **unique** indexes/constraints MUST match where upstream code targets them (`ON CONFLICT` targets, idempotent constraint drops in migrations).
- **DATALAYER-R9** (MUST) Boolean activity flags are `is_active boolean DEFAULT true` where upstream defines them.

### Migrations

- **DATALAYER-R10** (MUST) Migrations are managed **per module**: each module has its own migration files and its own tracking table named `mikro_orm_migrations_<sanitized module id>` (sanitizer: replace characters outside `[a-z0-9_]`, case-insensitive, with `_`). The tracking table has MikroORM's standard columns (`id`, `name`, `executed_at`). A port sharing a database with the Node runtime MUST read/write these same tables.
- **DATALAYER-R11** (MUST) Migration application iterates modules **alphabetically by module id**, applying each module's pending migrations one at a time in file order.
- **DATALAYER-R12** (MUST) The port ships real migration tooling (generate + apply) against Postgres. Generated migration names SHOULD follow `Migration<YYYYMMDDHHMMSS>_<moduleId>` so histories are mergeable with upstream's; the *recorded name* in the tracking table MUST match what was applied.
- **DATALAYER-R13** (SHOULD) Constraint drops in generated migrations are made idempotent (`drop constraint if exists`), and schema operations run with FK checks relaxed, mirroring upstream's `disableForeignKeys` behavior.
- **DATALAYER-R14** (MUST) `DATABASE_URL` is the canonical connection env var; startup fails with an error when it is unset.

### Multi-tenancy schema and scope resolution

- **DATALAYER-R15** (MUST) The `tenants` table: `id uuid PK`, `name text`, `is_active bool default true`, `created_at`, `updated_at`, `deleted_at`.
- **DATALAYER-R16** (MUST) The `organizations` table: `id uuid PK`, `tenant_id uuid` (FK→tenants), `name text`, `slug text null`, `logo_url text null`, `is_active bool default true`, materialized tree columns `parent_id uuid null`, `root_id uuid null`, `tree_path text null`, `depth int default 0`, `ancestor_ids jsonb default []`, `child_ids jsonb default []`, `descendant_ids jsonb default []`, timestamps, `deleted_at`; unique `organizations_tenant_slug_uniq (tenant_id, slug)`.
- **DATALAYER-R17** (MUST) Writes that change the org tree keep `parent_id`/`root_id`/`tree_path`/`depth`/`ancestor_ids`/`child_ids`/`descendant_ids` consistent — scope expansion reads `descendant_ids` directly and does no recursive queries.
- **DATALAYER-R18** (MUST) Request scope resolution produces `OrganizationScope = { selectedId, filterIds, allowedIds, tenantId }` from: JWT claims (`sub`, `tenantId`, `orgId`, `isSuperAdmin`), the user's RBAC-allowed organizations, and cookies `om_selected_org` / `om_selected_tenant`. The tenant-override cookie is honored **only** for super admins; other users are pinned to their token's tenant.
- **DATALAYER-R19** (MUST) Every selected/allowed organization id is expanded with its persisted `descendant_ids` (filtered by tenant and `deleted_at IS NULL`): selecting a parent org includes all descendants.
- **DATALAYER-R20** (MUST) `filterIds === null` means "no org restriction" (super admin / all orgs); `filterIds === []` means "access nothing". Ports MUST preserve this null-vs-empty distinction end to end.

### Multi-tenancy enforcement (application-level; no Postgres RLS)

- **DATALAYER-R21** (MUST) List/read routes with an empty org scope (`organizationIds === []`) return HTTP **200** with `{ "items": [], "total": 0, "page": ..., "pageSize": ..., "totalPages": 0 }` without querying the base table (and SHOULD log a forbidden-access event `organization_scope_empty`).
- **DATALAYER-R22** (MUST) The query engine requires `tenantId` on every call; a missing tenantId is a programming error that throws (surfacing as HTTP 500, not 403).
- **DATALAYER-R23** (MUST) When the target table has a `tenant_id` column, the engine adds `WHERE tenant_id = :tenantId`. When it has `organization_id` and an org scope was provided: `organization_id IN (:ids)`, with `OR organization_id IS NULL` only when the caller included a null/empty entry in `organizationIds`; ids and include-null both empty produce `WHERE 1 = 0`. `organizationIds` takes precedence over `organizationId`.
- **DATALAYER-R24** (MUST) The same tenant/org/soft-delete guards are applied to every join alias whose table has the columns, including `entity_indexes` and `search_tokens` subqueries. Tables lacking the columns get no guard (column-existence–driven).
- **DATALAYER-R25** (MUST) `omitAutomaticTenantOrgScope: true` disables the automatic guards for that query (the caller encodes visibility in filters).
- **DATALAYER-R26** (MUST) Command-level scope guards: a tenant mismatch or an organization outside the actor's allowed set is rejected with HTTP 403 `{ "error": "Forbidden" }`; cross-tenant/org entity relations are rejected with 403 `{ "error": "Cross-tenant relation forbidden" }`. `allowedIds === null` means unrestricted; super admins pass. When `OM_ENFORCE_ORG_SCOPE_STRICT` is unset/false, a completely unscoped call is allowed but logged.
- **DATALAYER-R27** (MUST) Command handlers additionally include `tenant_id`/`organization_id` (and `deleted_at IS NULL`) in every record-lookup where clause, returning 404 when the record exists but is out of scope. Scope filtering MUST NOT rely on the guards alone.
- **DATALAYER-R28** (MUST NOT) Do not introduce Postgres RLS or DB-level tenancy enforcement — upstream tooling (reindexers, CLI, migrations) connects without tenant context and would break.

### Custom fields & custom entities

- **DATALAYER-R29** (MUST) Reproduce these tables exactly (columns per the Contracts section): `custom_field_defs`, `custom_field_entity_configs`, `custom_entities`, `custom_entities_storage`, `custom_field_values`, `encryption_maps`.
- **DATALAYER-R30** (MUST) Custom field kinds are exactly `text|multiline|integer|float|boolean|select|currency|relation|attachment|dictionary` (stored in `custom_field_defs.kind` as text), with per-field config in `config_json`.
- **DATALAYER-R31** (MUST) EAV kind→column mapping: `text|select|currency|dictionary → value_text`, `multiline → value_multiline`, `integer → value_int`, `float → value_float`, `boolean → value_bool`; unknown/undefined kind falls back to JS-type inference (boolean→`value_bool`, integral number→`value_int`, other number→`value_float`, else `value_text`); encrypted fields always use `value_text`. Exactly one value column is populated per row.
- **DATALAYER-R32** (MUST) Multi-value custom-field writes are replace-semantics executed atomically: delete all rows for `(entity_id, record_id, organization_id, tenant_id, field_key)` then insert one row per element, in a single DB transaction (joining an ambient transaction when one exists).
- **DATALAYER-R33** (MUST) Field-definition scoping: on read, a def is visible when `is_active AND (tenant_id = :tenant OR tenant_id IS NULL)`; when several defs share a key, the winner is the most specific scope (tenant > org > global), ties broken by newest `updated_at`. Value rows match when `(organization_id = :org OR IS NULL) AND (tenant_id = :tenant OR IS NULL)`.
- **DATALAYER-R34** (MUST) `custom_field_values.record_id` and `entity_indexes.entity_id` / `custom_entities_storage.entity_id` are `text` (record ids of any PK type are stringified); joins cast the base uuid PK with `::text`.
- **DATALAYER-R35** (MUST) Doc-storage writes for ORM-backed (system) entity ids are rejected with HTTP 400 `{ "error": "Records are available for custom entities only", "code": "system_entity_records_blocked", "entityId": "<id>" }`. Read routing likewise never sends an ORM-backed id to doc storage (even if `custom_entities` rows exist for it), except under an explicit `forceCustomEntityStorage` flag.
- **DATALAYER-R36** (MUST) Doc-storage create: record ids that are missing, non-uuid, or one of the sentinels `create|new|null|undefined` (case-insensitive) are replaced with a server-generated uuid; value keys are normalized (`cf_x`/`cf:x` → `cf:x`); callers can never set `id`/`entity_id` inside `doc`; the stored `doc` always contains the record's own `id`. Upsert uses `ON CONFLICT (entity_type, entity_id, organization_id) DO UPDATE SET doc, updated_at = now(), deleted_at = NULL`. Update is a shallow merge of `doc`; delete is soft (`deleted_at = now()`) plus best-effort soft delete of matching `custom_field_values`.
- **DATALAYER-R37** (MUST) Modules can declare custom entities and field sets (upstream `ce.ts`); an installer syncs them per tenant (global specs once with `tenant_id NULL`, otherwise once per existing tenant), upserting `custom_entities` and `custom_field_defs` rows (organization_id NULL, tenant_id = scope), skipping system entity ids for the registry row. It SHOULD skip unchanged specs via a checksum cache (upstream key `custom-entities:v1:<scope>:<entityId>`).
- **DATALAYER-R38** (MUST) A per-record cap on distinct custom-field keys is enforced on write (upstream `MAX_CUSTOM_FIELD_KEYS_PER_RECORD`).

### Query engine

- **DATALAYER-R39** (MUST) Provide a single read abstraction `query(entityId, options) -> { items, page, pageSize, total, meta? }` supporting: `fields` (base columns and/or `cf:<key>`), `includeCustomFields` (boolean or key list), `filters` (op set `eq|ne|gt|gte|lt|lte|in|nin|like|ilike|exists`, plus Mongo-style `$eq/$ne/$gt/$gte/$lt/$lte/$in/$nin/$like/$ilike/$exists` objects and `$or` groups), `sort` (`{field, dir}` list), `page` (`{page, pageSize}`), `tenantId` (required), `organizationId`/`organizationIds`, `withDeleted`, `omitAutomaticTenantOrgScope`, `forceCustomEntityStorage`.
- **DATALAYER-R40** (MUST) Filter semantics: `eq`/`ne` with `null` → `IS NULL`/`IS NOT NULL`; `in`/`nin` wrap non-array values; `exists: true` → `IS NOT NULL`; `$or` groups combine as (AND within group) OR (across groups) inside the overall AND.
- **DATALAYER-R41** (MUST) Pagination defaults `page=1`, `pageSize=20`; `total` is computed by a separate count query before limit/offset; count uses `count(distinct base.id)` when joins may fan out, plain `count(*)` otherwise.
- **DATALAYER-R42** (MUST) Result items carry base columns under **snake_case** keys (raw column selection, no camelCase mapping) and custom fields under `cf:<key>` keys. Single-value custom fields are scalars; multi fields (per `config_json->>'multi'` on the def) are JSON arrays. On the EAV/basic path, scalar CF values are text-serialized (`"42"`, `"true"`); on the index/doc path, native JSON types are preserved.
- **DATALAYER-R43** (MUST) Entity-id→table resolution follows upstream's cascade (ORM metadata by PascalCase class name / `<Name>Entity`, then candidate tables `<module>_<name>`, pluralized forms, then naive pluralization with a warning). A strict variant returning null/absent for unregistered ids MUST be used at security-sensitive boundaries (reindexer, doc-storage guard).
- **DATALAYER-R44** (MUST) `like`/`ilike` filters are rewritten to `search_tokens` hash-conjunction subqueries when tokens exist for the entity+scope (all query-token hashes must match: `HAVING count(distinct token_hash) >= N`); plain SQL `like`/`ilike` applies otherwise. This is approximate token matching, and a port MUST keep both branches.
- **DATALAYER-R45** (MUST) A filter/sort on `organization_id` against a base table lacking that column resolves to `id` instead (upstream `resolveBaseColumn` quirk; required for `directory:organization` queries).
- **DATALAYER-R46** (MUST) Read routing cascade: (1) custom (non-ORM-backed) entity ids, or `forceCustomEntityStorage`, query `custom_entities_storage` (`id` ↔ `entity_id`, timestamps from real columns, other fields from `doc ->> field`); (2) missing base table or `omitAutomaticTenantOrgScope` → non-indexed fallback; (3) queries needing custom fields consult index coverage — no index rows → fallback; coverage gap → fallback **with** `meta.partialIndexWarning = { entity, entityLabel, baseCount, indexedCount, scope }` and an auto-reindex scheduled; (4) otherwise indexed path: base table LEFT JOIN `entity_indexes` on `(entity_type, entity_id = base.id::text [, organization_id, tenant_id, deleted_at IS NULL])`, custom fields read from `ei.doc ->> 'cf:<key>'`, doc-key sorting via `(ei.doc ->> field)`.
- **DATALAYER-R47** (SHOULD) A port MAY implement a single engine instead of two, but the indexed projection (`entity_indexes`) MUST still be populated (other modules read it) and the fallback + `partialIndexWarning` semantics MUST be preserved for coverage gaps.
- **DATALAYER-R48** (SHOULD) Column/table existence used for guard decisions is discovered at runtime (`information_schema`) with positive-only caching; a statically-schema'd port may hardcode, but MUST keep "guard only when the column exists" behavior.

### query_index maintenance (write side)

- **DATALAYER-R49** (MUST) Reproduce the tables `entity_indexes` (including the **stored generated column** `organization_id_coalesced uuid GENERATED ALWAYS AS (coalesce(organization_id, '00000000-0000-0000-0000-000000000000'::uuid)) STORED` and unique index `entity_indexes_type_entity_org_coalesced_unique (entity_type, entity_id, organization_id_coalesced)`), `entity_index_jobs`, `entity_index_coverage`, `indexer_error_logs`, `indexer_status_logs`, `search_tokens` — names and columns per the analysis.
- **DATALAYER-R50** (MUST) Every ORM-entity write emits (a) the CRUD event `<module>.<entity>.created|updated|deleted` with payload `{ id, organizationId, tenantId }` and (b) `query_index.upsert_one` / `query_index.delete_one` with payload `{ entityType, recordId, organizationId, tenantId, crudAction, coverageBaseDelta? }`. The index upsert/delete is **awaited before the write returns** (read-your-writes on `entity_indexes`); search-token rebuilds are deferred/eventually consistent.
- **DATALAYER-R51** (MUST) Index-maintenance failures are logged to `indexer_error_logs` and MUST NOT fail the originating write.
- **DATALAYER-R52** (MUST) The index document (`entity_indexes.doc`) is the base row with **snake_case keys**, merged with `cf:<key>` entries (scoped org/tenant match-or-NULL; single scalar, multi array) and `l10n:<locale>:<field>` translation entries, plus an aggregate search field. Upserts target `ON CONFLICT (entity_type, entity_id, organization_id_coalesced)` and reset `deleted_at` to NULL (revive).
- **DATALAYER-R53** (MUST) Deleting a record **hard-deletes** its `entity_indexes` row(s) and `search_tokens` rows (despite `deleted_at` existing on the table); coverage math depends on removal/revival.
- **DATALAYER-R54** (MUST) Coverage counters (`entity_index_coverage`, unique on `(entity_type, tenant_id, organization_id, with_deleted)`) are adjusted incrementally per write (`coverageBaseDelta` +1/−1) and refreshed periodically (throttled, default 5 min).
- **DATALAYER-R55** (SHOULD) Full reindex runs as partitioned (default 5), batched (500) jobs tracked in `entity_index_jobs` via atomic upsert on the coalesced-scope unique index, with heartbeats and `status` in `reindexing|purging`. `query_index.reindex` / `query_index.purge` events are persistent (queue-backed); `upsert_one`/`delete_one` are not.
- **DATALAYER-R56** (SHOULD) Index-maintenance handlers run on a forked/independent unit of work so they cannot drop or flush the caller's pending changes.

### Commands (write pipeline)

- **DATALAYER-R57** (MUST) All mutating API operations execute as named commands (`'<module>.<domain>.<action>'`, e.g. `customers.tags.create`) through a command bus whose pipeline is: before-interceptors → `prepare` (before-snapshot) → `execute` (or `redo` on replay) → `captureAfter` (after-snapshot) → `buildLog` → persist an action-log row (input wrapped in a `{ "__redoInput": <input>, ... }` envelope under the command payload, with auto-generated undo token and auto-diffed changes) → after-interceptors → CRUD cache invalidation → flush queued side effects (CRUD events + index maintenance).
- **DATALAYER-R58** (MUST) Side effects (events, index upserts) are emitted **after** the command's database work, never inside its transaction.
- **DATALAYER-R59** (MUST) Undo claims the action-log row atomically (status CAS `done → undoing`), runs the handler's `undo`, marks it undone, and writes an inverse trace log. Redo re-executes from the stored `__redoInput`, reusing original record ids.
- **DATALAYER-R60** (MUST) Handlers validate input first, then apply scope guards (R26), then operate on a fresh unit of work — unless a shared transactional unit (`ctx.transactionalEm` equivalent) is supplied by a composing command, which MUST be reused so composed commands share one transaction.
- **DATALAYER-R61** (MUST) Multi-phase writes offer an atomic-flush primitive: each phase's changes are flushed at the phase boundary, and with `transaction: true` all phases commit in one `BEGIN/COMMIT` (joining an ambient transaction rather than nesting). Commands are NOT transactional by default; wrap per handler, not globally.
- **DATALAYER-R62** (MUST) Duplicate detection is an explicit pre-write query returning HTTP 409 (e.g. `{ "error": "Tag slug already exists" }`), not DB-constraint-error mapping. The DB unique constraints still exist as a race backstop.
- **DATALAYER-R63** (MUST) Command error statuses: 400 validation (`{ "error": "Validation failed", "fields": {...} }`) and invalid relations, 403 scope violations, 404 missing/out-of-scope records, 409 duplicates, 422 for domain-specific reference errors where upstream uses it.

| `OM_ORG_SCOPE_CACHE_TTL_MS` | `0` (off) | cross-request org-scope cache TTL |

### Cookies

`om_selected_org` (selected organization uuid or all-orgs sentinel), `om_selected_tenant` (super-admin-only tenant override).

### Cross-cutting DDL (byte-identical names)

Tables: `tenants`, `organizations`, `custom_field_defs`, `custom_field_entity_configs`, `custom_entities`, `custom_entities_storage`, `custom_field_values`, `encryption_maps`, `entity_indexes`, `entity_index_jobs`, `entity_index_coverage`, `indexer_error_logs`, `indexer_status_logs`, `search_tokens`, `action_logs`, per-module business tables, and `mikro_orm_migrations_<module>` tracking tables.

Key column sets (full detail in the analysis document):

- `custom_field_defs(id, entity_id text, organization_id uuid null, tenant_id uuid null, key text, kind text, config_json json null, is_active bool default true, created_at, updated_at, deleted_at)`; unique/lookup indexes `cf_defs_entity_tenant_org_idx`, `cf_defs_entity_key_scope_idx(entity_id,key,tenant_id,organization_id)`, etc.
- `custom_field_values(id, entity_id text, record_id text, organization_id uuid null, tenant_id uuid null, field_key text, value_text text, value_multiline text, value_int int, value_float float, value_bool boolean, created_at, deleted_at)` — **no `updated_at`**; indexes `cf_values_entity_record_tenant_idx(entity_id,record_id,tenant_id)`, `cf_values_entity_record_field_idx(field_key)`.
- `custom_entities_storage(id, entity_type text, entity_id text, organization_id uuid null, tenant_id uuid null, doc json, created_at, updated_at, deleted_at)`; unique `custom_entities_storage_unique_idx(entity_type, entity_id, organization_id)`.
- `custom_entities(id, entity_id text, label, description, label_field, default_editor, show_in_sidebar bool default false, organization_id/tenant_id uuid null, is_active, timestamps, deleted_at)`; unique `custom_entities_unique_idx(entity_id, organization_id, tenant_id)`.
- `entity_indexes(id, entity_type text, entity_id text, organization_id uuid null, organization_id_coalesced uuid GENERATED ALWAYS AS (coalesce(organization_id,'00000000-0000-0000-0000-000000000000'::uuid)) STORED, tenant_id uuid null, doc json, embedding json null, index_version int default 1, created_at, updated_at, deleted_at)`; unique `entity_indexes_type_entity_org_coalesced_unique(entity_type, entity_id, organization_id_coalesced)`.
- `search_tokens(entity_type, entity_id, organization_id/tenant_id null, field, token_hash, token null, created_at)`; index `search_tokens_lookup_idx(entity_type,field,token_hash,tenant_id,organization_id)`.
- `entity_index_coverage(entity_type, tenant_id, organization_id, with_deleted)` unique → `base_count, indexed_count, vector_indexed_count, refreshed_at`.
- `entity_index_jobs`: coalesced-scope raw unique index over `(entity_type, coalesce(org, zero-uuid), coalesce(tenant, zero-uuid), coalesce(partition_index,-1), coalesce(partition_count,-1))`.
- `organizations` unique: `organizations_tenant_slug_uniq(tenant_id, slug)`.

### Event names & payloads (data-layer originated)

- `<module>.<entity>.created|updated|deleted` → `{ "id": "...", "organizationId": "...", "tenantId": "..." }`
- `query_index.upsert_one` / `query_index.delete_one` (persistent: false) → `{ "entityType", "recordId", "organizationId", "tenantId", "crudAction"?, "coverageBaseDelta"?, "coverageIndexDelta"?, "suppressCoverage"?, "coverageDelayMs"?, "searchTokenDoc"?, "syncOrigin"? }`
- `query_index.reindex` / `query_index.purge` (persistent: true)
- `query_index.coverage.refresh` → `{ "entityType", "tenantId", "organizationId", "delayMs" }`
- `query_index.vectorize_one`; `search.index_record` → `{ "entityId", "recordId", "organizationId", "tenantId" }`

### HTTP shapes (data-layer originated)

- List: `200 { "items": [...], "total": n, "page": p, "pageSize": s, "totalPages": t }` — items with snake_case base keys, `cf:<key>` custom-field keys (aliases sanitized to `cf_<key>` where the SQL layer requires `[a-zA-Z0-9_]`), multi CFs as arrays. Empty org scope → same shape with `items: []`, `total: 0`, HTTP 200.
- `400 { "error": "Validation failed", "fields": {...} }`
- `400 { "error": "Records are available for custom entities only", "code": "system_entity_records_blocked", "entityId": "<id>" }`
- `403 { "error": "Forbidden" }` / `403 { "error": "Cross-tenant relation forbidden" }`
- `404 { "error": "<Thing> not found" }`
- `409 { "error": "<Thing> already exists" }` (e.g. `Tag slug already exists`)
- `meta.partialIndexWarning`: `{ "entity", "entityLabel", "baseCount", "indexedCount", "scope" }` attached to list results on coverage-gap fallback.

### Selector syntax

- Entity id: `'<module>:<entity>'` per `/^[a-z][a-z0-9_]*:[a-z][a-z0-9_]*$/`.
- Custom-field selector: `cf:<key>`; translation selector: `l10n:<locale>:<field>`; doc keys in `entity_indexes.doc` / `custom_entities_storage.doc` use the same prefixes.

## Concept mapping

## Allowed deviations

Welcome (document each as a decision record):

- **ORM/data-access choice** — SQLAlchemy, EF Core, sqlc/GORM, etc. — as long as generated DDL, timestamps-in-app, and DB-side uuid defaults match.
- **Validation library** — pydantic, FluentValidation, go-playground/validator — as long as error responses keep the 400 `{ error: 'Validation failed', fields }` shape and equivalent rules.
- **Migration tool** — Alembic, EF migrations, golang-migrate — provided per-module tracking tables, alphabetical module order, and idempotent constraint drops are honored (R10–R13). Snapshot/diff internals are free.
- **Single query engine** instead of Basic+Hybrid, per R47.
- **Static schema knowledge** instead of `information_schema` probing, per R48.
- **Internal code structure** — no need to mirror file layout, class names, or the Kysely/MikroORM split; helper *behavior* (Helpers to mirror table in the analysis) is what's contractual.
- **Generated-column workaround** — if the schema tool cannot express `organization_id_coalesced` as a stored generated column, an expression unique index + matching `ON CONFLICT` expression target is acceptable (the pattern `entity_index_jobs` already uses), provided Node and the port can both upsert without conflict-target errors.

MUST NOT change:

- Any table/column/unique-index name or column type listed in Contracts; `record_id`/`entity_id` staying `text`.
- Null-vs-empty org-scope semantics (R20, R21, R23) and the mandatory `tenantId` throw (R22).
- Soft-delete defaults, and the hard-delete of `entity_indexes` rows on record deletion (R53).
- Read-your-writes on the index projection (R50) and "index errors never fail the write" (R51).
- HTTP status codes and JSON error bodies (R63, Contracts).
- Event names/payloads and their persistent/non-persistent classification.
- The system-entity doc-storage guard, both write- and read-side (R35).
- App-side timestamp generation (no DB triggers) and DB-side uuid PK defaults.

## Verification

`om-verify-parity` checks these requirements by running the port against the same Postgres/Redis as the reference Node deployment:

1. **Schema diff**: run the port's migrations on a clean database; dump `information_schema` (tables, columns, types, nullability, defaults, generated columns) and `pg_indexes` and diff against a dump produced by upstream `mercato db migrate`. R1–R9, R15–R16, R29, R49 fail on any name/type mismatch in contract tables. Verify `mikro_orm_migrations_<module>` tables exist per module and record applied names (R10–R12).
2. **Cross-runtime write/read**: create records via the Node reference, read them via the port (and vice versa) with the same JWT + cookies; list responses must be byte-comparable JSON (snake_case keys, `cf:*` keys, scalar/array CF shapes, pagination envelope) — R39–R42, R46.
3. **Tenancy matrix**: fixture with 2 tenants, org tree with descendants; assert (a) parent-org selection returns child-org rows (R19), (b) empty scope returns 200 empty envelope (R21), (c) cross-tenant command → 403 `{"error":"Forbidden"}` and cross-tenant relation → 403 `{"error":"Cross-tenant relation forbidden"}` (R26), (d) out-of-scope record id → 404 (R27), (e) query without tenantId → 500 (R22).
4. **Custom-field parity**: define defs of every kind, write single + multi values via the port, inspect `custom_field_values` rows directly (exactly one value column populated, correct column per kind, multi replace leaves no stale rows) — R31–R32; read back via Node reference and compare shapes (R42).
5. **Doc-storage guard**: POST a record for an ORM-backed entity id → 400 with `code: system_entity_records_blocked` (R35); sentinel record ids (`new`, `create`, ...) get server uuids and `doc.id` is set (R36); upsert conflict target works against the live unique index.
6. **Index projection**: after each port-side create/update/delete, immediately SELECT `entity_indexes` — row present/updated with snake_case + `cf:*` doc before the HTTP response completed (read-your-writes, R50); after delete, projection row is gone (R53); force an index error (e.g. drop grant) and confirm the write still succeeds and `indexer_error_logs` has a row (R51). Coverage-gap fixture returns `meta.partialIndexWarning` with the exact key set (R46).
7. **Command semantics**: duplicate create → 409 with upstream body (R62); action_logs row contains the `__redoInput` envelope; undo → record restored with original id; redo reuses stored input (R57, R59). Multi-phase command interrupted mid-phase (fault injection) leaves no partial commit when transactional (R61).
8. **Event capture**: subscribe a probe to the shared bus; each write emits the CRUD event and `query_index.upsert_one` with the exact payload keys (R50, Contracts). Persistent vs non-persistent classification checked against queue contents (spec 04 harness).
